# 요구사항 문서

## 소개

올리브영 검색 결과 페이지 URL을 입력하면, 해당 검색 결과에 포함된 각 상품의 리뷰를 크롤링하여 카테고리별로 정리하고 시각화하는 웹 애플리케이션이다. GitHub Pages를 통해 배포되며, 사용자가 URL을 입력하면 정리된 리뷰 데이터를 한눈에 확인할 수 있는 페이지를 생성한다.

## 용어 정의

- **Crawler**: 올리브영 웹사이트에서 상품 목록과 리뷰 데이터를 수집하는 모듈
- **Search_URL**: 올리브영 검색 결과 페이지의 전체 URL (예: `https://www.oliveyoung.co.kr/store/search/getSearchMain.do?...`)
- **Product_Card**: 검색 결과 페이지에서 표시되는 개별 상품 항목
- **Review_Page**: 각 상품의 리뷰가 표시되는 상세 페이지
- **Review_Data**: 리뷰 작성자, 평점, 본문, 작성일, 피부타입, 연령대 등의 정보를 포함하는 데이터 구조
- **Category**: 리뷰를 분류하는 기준 (평점, 피부타입, 연령대, 키워드 등)
- **Visualizer**: 수집된 리뷰 데이터를 차트 및 요약 형태로 시각화하는 모듈
- **URL_Validator**: 입력된 URL이 올리브영 검색 결과 페이지인지 검증하는 모듈
- **Web_App**: 사용자 인터페이스를 제공하는 프론트엔드 웹 애플리케이션

## 요구사항

### 요구사항 1: URL 입력 및 검증

**사용자 스토리:** 사용자로서, 올리브영 검색 결과 URL을 입력하여 해당 검색 결과의 리뷰를 수집하고 싶다.

#### 인수 조건

1. THE Web_App SHALL 올리브영 검색 결과 URL을 입력할 수 있는 입력 필드를 제공한다
2. WHEN 사용자가 URL을 입력하고 제출하면, THE URL_Validator SHALL 해당 URL이 `oliveyoung.co.kr` 도메인의 검색 결과 페이지인지 검증한다
3. WHEN 유효한 올리브영 검색 URL이 입력되면, THE URL_Validator SHALL 해당 URL에 접근 가능한지 확인한다
4. IF 입력된 URL이 올리브영 검색 결과 페이지 형식이 아닌 경우, THEN THE Web_App SHALL "올리브영 검색 결과 페이지 URL만 입력 가능합니다"라는 오류 메시지를 표시한다
5. IF 입력된 URL에 접근할 수 없는 경우, THEN THE Web_App SHALL "해당 URL에 접근할 수 없습니다. URL을 확인해 주세요"라는 오류 메시지를 표시한다

### 요구사항 2: 상품 목록 크롤링

**사용자 스토리:** 사용자로서, 검색 결과에 포함된 모든 상품의 정보를 자동으로 수집하고 싶다.

#### 인수 조건

1. WHEN 유효한 검색 URL이 확인되면, THE Crawler SHALL 검색 결과 페이지에서 모든 Product_Card의 상품명, 브랜드명, 가격, 상품 상세 페이지 URL을 추출한다
2. WHEN 검색 결과가 여러 페이지에 걸쳐 있는 경우, THE Crawler SHALL 모든 페이지의 상품을 순차적으로 수집한다
3. THE Crawler SHALL 수집된 상품 수를 사용자에게 실시간으로 표시한다
4. IF 특정 상품 페이지에 접근할 수 없는 경우, THEN THE Crawler SHALL 해당 상품을 건너뛰고 나머지 상품의 크롤링을 계속한다

### 요구사항 3: 리뷰 데이터 크롤링

**사용자 스토리:** 사용자로서, 각 상품의 리뷰를 상세하게 수집하여 분석에 활용하고 싶다.

#### 인수 조건

1. WHEN 상품 목록 수집이 완료되면, THE Crawler SHALL 각 상품의 Review_Page에서 리뷰 데이터를 수집한다
2. THE Crawler SHALL 각 리뷰에서 평점, 작성자 닉네임, 작성일, 리뷰 본문, 피부타입, 연령대 정보를 추출한다
3. WHEN 리뷰가 여러 페이지에 걸쳐 있는 경우, THE Crawler SHALL 최대 10페이지까지의 리뷰를 수집한다
4. THE Web_App SHALL 크롤링 진행 상황을 프로그레스 바로 표시한다
5. IF 리뷰 페이지 로딩 중 오류가 발생한 경우, THEN THE Crawler SHALL 최대 3회까지 재시도한 후 실패 시 해당 상품의 수집된 리뷰만 저장한다

### 요구사항 4: 리뷰 카테고리 분류

**사용자 스토리:** 사용자로서, 수집된 리뷰를 다양한 기준으로 분류하여 한눈에 파악하고 싶다.

#### 인수 조건

1. WHEN 리뷰 데이터 수집이 완료되면, THE Crawler SHALL 리뷰를 평점별(1~5점)로 분류한다
2. WHEN 리뷰 데이터 수집이 완료되면, THE Crawler SHALL 리뷰를 피부타입별(건성, 지성, 복합성, 민감성 등)로 분류한다
3. WHEN 리뷰 데이터 수집이 완료되면, THE Crawler SHALL 리뷰를 연령대별(10대, 20대, 30대, 40대, 50대 이상)로 분류한다
4. THE Crawler SHALL 리뷰 본문에서 자주 등장하는 키워드를 추출하여 키워드별로 분류한다
5. THE Crawler SHALL 각 상품별로 분류된 리뷰 데이터를 JSON 형식으로 구조화한다

### 요구사항 5: 리뷰 데이터 시각화

**사용자 스토리:** 사용자로서, 수집된 리뷰 데이터를 차트와 요약 형태로 한눈에 확인하고 싶다.

#### 인수 조건

1. THE Visualizer SHALL 각 상품의 평점 분포를 막대 차트로 표시한다
2. THE Visualizer SHALL 피부타입별 리뷰 비율을 원형 차트로 표시한다
3. THE Visualizer SHALL 연령대별 리뷰 비율을 원형 차트로 표시한다
4. THE Visualizer SHALL 자주 등장하는 키워드를 워드 클라우드 형태로 표시한다
5. THE Visualizer SHALL 각 상품의 전체 평균 평점과 총 리뷰 수를 요약 카드로 표시한다
6. THE Visualizer SHALL 검색 결과 전체 상품의 평균 평점 순위를 테이블로 표시한다

### 요구사항 6: 웹 애플리케이션 UI

**사용자 스토리:** 사용자로서, 직관적인 인터페이스를 통해 쉽게 리뷰 데이터를 탐색하고 싶다.

#### 인수 조건

1. THE Web_App SHALL 반응형 디자인을 적용하여 데스크톱과 모바일에서 모두 사용 가능하도록 한다
2. THE Web_App SHALL 상품 목록을 카드 형태로 표시하며, 각 카드를 클릭하면 해당 상품의 상세 리뷰 분석 페이지로 이동한다
3. THE Web_App SHALL 상품 목록을 평점순, 리뷰 수순, 가격순으로 정렬하는 기능을 제공한다
4. THE Web_App SHALL 특정 카테고리(피부타입, 연령대)로 리뷰를 필터링하는 기능을 제공한다
5. WHILE 크롤링이 진행 중인 상태에서, THE Web_App SHALL 현재 진행 상황과 예상 소요 시간을 표시한다

### 요구사항 7: 배포 및 접근성

**사용자 스토리:** 사용자로서, 별도의 설치 없이 웹 브라우저에서 바로 서비스를 이용하고 싶다.

#### 인수 조건

1. THE Web_App SHALL GitHub Pages 또는 유사한 정적 호스팅 서비스를 통해 배포된다
2. THE Web_App SHALL 별도의 로그인 없이 URL 입력만으로 서비스를 이용할 수 있도록 한다
3. THE Web_App SHALL 최신 버전의 Chrome, Firefox, Safari, Edge 브라우저에서 정상 동작한다
4. IF 크롤링 서버에 접근할 수 없는 경우, THEN THE Web_App SHALL "서비스에 일시적으로 접근할 수 없습니다. 잠시 후 다시 시도해 주세요"라는 메시지를 표시한다

### 요구사항 8: 크롤링 제한 및 안정성

**사용자 스토리:** 개발자로서, 올리브영 서버에 과도한 부하를 주지 않으면서 안정적으로 데이터를 수집하고 싶다.

#### 인수 조건

1. THE Crawler SHALL 각 페이지 요청 사이에 1초 이상의 간격을 두고 요청한다
2. THE Crawler SHALL 동시에 최대 3개의 요청만 병렬로 처리한다
3. IF 올리브영 서버로부터 429(Too Many Requests) 응답을 받은 경우, THEN THE Crawler SHALL 30초 대기 후 재시도한다
4. IF 올리브영 서버로부터 5회 연속 오류 응답을 받은 경우, THEN THE Crawler SHALL 크롤링을 중단하고 사용자에게 "서버 접근이 제한되었습니다. 잠시 후 다시 시도해 주세요"라는 메시지를 표시한다
5. THE Crawler SHALL 한 번의 크롤링 세션에서 최대 100개 상품까지만 처리한다
